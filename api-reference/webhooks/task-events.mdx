---
title: "Task Events"
description: "Webhook payloads for task create, assign, and finish events"
---

Task webhooks are triggered when tasks are created, assigned, or completed in MileApp. The webhook payload contains task details including flow fields configured in your workflow.

<Note>
In this example, the flow called `Delivery` has 2 custom fields: **invoice** and **deliveryCoordinate**. Your actual payload will include the fields defined in your flow.
</Note>

## Event Types

| Event | Trigger | Status |
|-------|---------|--------|
| **onCreate** | Task is created | `UNASSIGNED` |
| **onAssign** | Task is assigned to a worker | `ONGOING` |
| **onFinish** | Task is completed | `DONE` |

## Payload Fields

### Base Fields

| Field | Type | Description |
|-------|------|-------------|
| `_id` | string | Unique identifier of the task |
| `organizationId` | string | Organization ID associated with the task |
| `hubId` | string | Hub ID associated with the task |
| `flowId` | string | Flow ID associated with the task |
| `flow` | string | Flow name |
| `status` | string | Task status: `UNASSIGNED`, `ONGOING`, or `DONE` |
| `startTime` | string | Expected start time (ISO 8601) |
| `endTime` | string | Expected end time (ISO 8601) |
| `createdTime` | string | Task creation timestamp (ISO 8601) |
| `updatedTime` | string | Last update timestamp (ISO 8601) |
| `createdBy` | string | Name of the user who created the task |
| `createdFrom` | string | Source of task creation: `WEB` or `API` |
| `orderIndex` | integer | Order index for task sorting |

### Hub Object

| Field | Type | Description |
|-------|------|-------------|
| `_id` | string | Hub unique identifier |
| `name` | string | Hub name |
| `lat` | number | Hub latitude |
| `lng` | number | Hub longitude |
| `deleteAccess` | boolean | Whether delete access is allowed |

### Assignment Fields (onAssign, onFinish)

| Field | Type | Description |
|-------|------|-------------|
| `assignedBy` | string | Name of the person who assigned the task |
| `assignee` | array | Array of assignee email addresses |
| `assignedTo` | object | Details of the assigned worker |
| `assignedTime` | string | Assignment timestamp (ISO 8601) |

### Completion Fields (onFinish)

| Field | Type | Description |
|-------|------|-------------|
| `doneBy` | string | Name of the worker who completed the task |
| `doneTime` | string | Completion timestamp (ISO 8601) |

## Response

Your endpoint must return an HTTP 200 status code to acknowledge receipt:

```json
{
  "status": "received"
}
```
